## 二、🔴 严重差异

### 差异1：文档声称 IOA 有 `block_ip(ip, reason, duration)` 三参数方法，实际只有两参数

**文档**（3.1 节）：
```
- block_ip(ip, reason, duration) — IP 黑名单管理
```

**代码**（`ioa_engine.py:329`）：
```python
def block_ip(self, ip: str, reason: str) -> None:
    self._blocked_ips.add(ip)
    self._audit.log_alert(f"[IOA] IP封禁: {ip} | 原因: {reason}")
```

**差异**：代码中 `block_ip` 只接受 `(ip, reason)` 两个参数，无 `duration` 参数。封禁是永久的（存入 set），需通过 `/api/security/threats/unblock-ip` API 手动解封。

**影响**：文档暗示有自动过期封禁，实际无此功能。

---

### 差异2：文档声称 `AgentGuard` 有"30秒窗口内重复拒绝"和"单字段上限 2000 字符"

**文档**（3.2 节）：
- NPC反重放保护：**SHA256 哈希生成决策指纹，30秒窗口内重复拒绝**
- 玩家参数长度限制：**单字段上限 2000 字符**

**代码**（`agent_guard.py`）：
- 反重放（`validate_npc_decision` 第191-195行）：`recent_count = sum(1 for f in self._decision_fingerprints if f == fp)` — **无时间窗口**，纯计数 `> 2` 触发（10000条deque全局去重）
- 参数长度限制（`validate_player_action` 第261行）：`if len(val_str) > 10000` — 上限是 **10000 字符**，不是 2000

**影响**：
- 反重放机制完全不同于文档描述（无30秒窗口，而是全局指纹去重+计数）
- 参数长度限制是文档声称的 5 倍

### 差异3：文档声称 `validate_npc_decision` 有"每秒最多 5 次"频率限制

**文档**（3.2 节）：
```
频率检测：单个 NPC 每秒最多 5 次决策
```

**代码**（`agent_guard.py:_check_action_frequency` 第292-305行）：
```python
if n > 30: return 90.0   # 1分钟内30次 = 每秒0.5次
```

**差异**：频率检测窗口是 **60秒**，阈值分别为 8/15/30 次。文档声称的"每秒5次"=300次/分钟，与代码差异巨大。实际限制严格得多（约 0.5次/秒触发最高分）。
