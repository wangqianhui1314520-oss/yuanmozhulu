## 第二轮：后端核心逻辑（按子系统）

### 2.1 回合引擎 (round_engine.py)

#### Bug #1 [P0] 回合锁超时后死锁
- **现象**：`asyncio.wait_for` 超时时 `acquire()` 可能已成功获取锁，但 `finally` 中不释放
- **位置**：`round_engine.py` 第123-130行
- **复现**：AI 调用超过 300 秒超时 → 抛出 TimeoutError → 锁被永久持有
- **期望**：超时后应在 finally 中安全释放锁
- **影响**：所有后续回合调用永久阻塞

#### Bug #2 [P0] 阶段1校验失败后阶段2盲目执行
- **现象**：`valid_indices` 为空集合时，`if valid_indices and ...` 短路为 False，所有指令被执行
- **位置**：`round_engine.py` 第640行
- **复现**：阶段1抛异常 → `valid_indices` = `set()` → 所有指令通过
- **期望**：阶段1失败时阶段2应全部跳过
- **影响**：未校验的危险指令被执行

#### Bug #3 [P0] 裸 `except Exception` 静默吞异常
- **现象**：`_resp_tracker.get_stats()` 异常被静默吞掉，无日志
- **位置**：`round_engine.py` 第181-183行
- **复现**：responsibility tracker 内部错误 → 被静默替换为 `{"error": "unavailable"}`
- **期望**：至少记录日志
- **影响**：关键错误信息丢失，难以排查

#### Bug #4 [P1] 回合号与时间不同步
- **现象**：回合号在第137行递增，但如果所有阶段失败，时间仍推进但数据未更新
- **位置**：`round_engine.py` 第137行 vs 第174行
- **复现**：系统性错误导致所有阶段失败
- **期望**：阶段失败时应回滚回合号
- **影响**：数据不一致

#### Bug #5 [P1] 操作锁对未知 action 返回 True
- **现象**：`_acquire_operation_lock` 对未识别 action 返回 True
- **位置**：`round_engine.py` 第356行
- **复现**：传入不存在的 action → 锁检查形同虚设
- **期望**：未知 action 应拒绝或至少警告
- **影响**：锁机制被绕过

#### Bug #6 [P2] 事件日志仅检查最近20条
- **现象**：`events_log[-20:]` 可能遗漏重要事件
- **位置**：`round_engine.py` 第1323行
- **复现**：一回合产生超过20条 major 事件
- **期望**：检查全量事件或按严重度筛选
- **影响**：governance_logs 丢失重要事件

#### Bug #7 [P2] recruit 操作不检查 tile 是否为 None
- **现象**：阶段1失败后 `tile` 可能为 None，直接访问 `tile.tile_type` 崩溃
- **位置**：`round_engine.py` 第687-689行
- **复现**：阶段1校验被跳过 + recruit 指令的 tile 不存在
- **期望**：防御性检查 tile 非空
- **影响**：AttributeError 崩溃

### 2.2 行军结算 (settle_engine.py)

#### Bug #8 [P0] 谍报成功率超过100%
- **现象**：`base_success = 0.75 + network.infiltration * 0.08`，渗透度达3.125%时成功率即超100%
- **位置**：`settle_engine.py` 第1759行

- **期望**：`min(base_success, 0.95)` 或类似上限
- **影响**：谍报系统完全失去挑战性

#### Bug #9 [P1] 战败时兵力退还缺少事务性回滚
