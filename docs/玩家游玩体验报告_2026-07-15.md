## 二、🔴 严重问题（6项）

### 🔴 #1 所有势力领地为空（tiles=[]），初始化严重缺陷

**现象**: 游戏初始化后，`/api/game/status` 返回的 `world_state.factions` 中，所有9个势力的 `tiles` 字段均为空数组 `[]`，`tile_count` 也为 0 或 1。

**具体数据**:
- 元廷: tiles=0, troops=-1
- 朱元璋: tiles=0, troops=0
- 陈友谅: tiles=0, troops=-1
- ...全部9个势力 tiles 均为空

**对比**: `init` 接口返回的 `player_faction.tile_count` 是 11（正确），但 `world_state` 中的 `faction.tiles` 列表为空。

**影响**: 这是最致命的问题——势力没有领地，导致：
- 所有需要指定地点的操作（行军/征兵/建造/侦查）找不到目标地块
- `troops=-1` 表示兵力数据异常
- 游戏实际上处于"无领土幽灵状态"

**推测原因**: `init` 和 `status` 两个端点返回的 `world_state` 数据不一致。`init` 返回了 `player_faction` 和 `factions` 两个独立对象，但 `status` 可能走了不同的序列化路径。

---

### 🔴 #2 地块ID体系不一致：tile_yingtian vs "18,12"

**现象**: 
- 势力配置中 `capital_tile: "tile_yingtian"`（语义化ID）
- 地图 `/api/map/tiles` 返回的地块 key 是坐标格式：`"15,21"`, `"18,17"` 等
- `/api/map/tile/tile_yingtian` 返回空/错误

**数据**:
```
势力首都配置: capital_tile = "tile_yingtian"
地图实际地块ID: ['15,21', '18,17', '18,26', '27,4', ...]  (纯坐标格式)
搜索 tile_yingtian: 0 个结果
```

**影响**: 
- 点击应天府查看详情 → 失败
- 右键地块快速下指令 → 因为用的是 tile_yingtian，找不到对应坐标
- 圣旨中"从应天府调拨..." → AI 解析到 tile_yingtian，但地图上没有这个ID

---

### 🔴 #3 初始化后资源数据丢失

**现象**: `init` 返回的初始数据正确（库银8000、粮草4000、声望40、军械80），但后续 `status` 查询时大量字段变为 None/空。

| 字段 | init 值 | status 值 |
|------|---------|-----------|
| treasury(库银) | 8000 | 2272 → 后续变为 None |
| reputation(声望) | 40 | None |
| arms(军械) | 80 | None |
| popular_support(民心) | 无此字段 | None |

**注意**: init 返回的 `player_faction` 在 `data` 顶层（不在 `world_state` 中），但 `status` 返回的是 `world_state.factions`，两者结构不同。`status` 中 treasury 有值(2272)但 `silver` 为 None——说明字段名也不一致。

---

### 🔴 #4 征兵永远失败：军械不足的死循环

**现象**: 连续多个回合尝试征兵，全部因"军械不足"失败。

```
第0回合: 征兵3000 → 军械不足（需要1000，现有80）
第2回合: 征兵2000 → 军械不足（需要666，现有80）
第4回合: 征兵1000 → 仍可能失败
```

**分析**: 
- 初始军械只有80，征兵3000需要1000军械
- 每回合军械不自动增长
- 建造军械所需要先有军械（鸡生蛋问题）
- 玩家陷入"无军械→无法征兵→无法扩张→无资源造军械所"的死循环

---

### 🔴 #5 所有势力 troops=-1（数据异常）

**现象**: 8个NPC势力兵力均为 -1。

```
元廷: troops=-1
陈友谅: troops=-1
张士诚: troops=-1
...全部8个NPC势力
```

这导致：回合结算时所有AI势力处于"已灭亡"状态，AI推演无法正常进行。

---

### 🔴 #6 train_troops 逻辑错误："训练数量无效（驻军2943）"

**现象**: 圣旨"征兵三千，加紧训练"被解析为 recruit + train_troops 两个指令，但 train_troops 失败，报错"训练数量无效（驻军2943）"。

**分析**: 
- 玩家 total_troops=2943，但 `troops` 字段（可调动兵力）= 0
- `train_troops` 检查时用了 total_troops(2943)，但无法从中分出可训练部队
- 这是个数据不一致导致的连锁问题

## 三、🟡 中等问题（7项）

### 🟡 #7 mode 字段位置不一致

`init` 返回中，`mode` 在 `data.mode_info.mode` 而非 `data.mode`：
```json
{
  "data": {
    "mode": null,           // ← 顶层为 null
    "mode_info": {
      "mode": "player_turn",
      "mode_label": "君主亲政"
    }
  }
}
```
前端如果读取 `data.mode` 会得到 null。

### 🟡 #8 战略建议内容空洞

`/api/agent/strategic-advice` 返回 `response: "时局未明，容臣思之。"`

这是一句敷衍回复。A1谋策阁的战略分析应该是核心卖点之一，但实际返回了兜底/fallback文本，说明LLM调用可能超时或未正确传入足够的上下文。

### 🟡 #9 操作锁过于严格

同一回合内多次下圣旨，同类型操作会被锁住：
```
build - 地块26,15的本回合已执行此类操作
```
文档描述"同类型指令每回合只生效一次"是设计如此，但错误提示不够友好，未告知玩家如何规避。

### 🟡 #10 存档列表字段名不一致

- 文档使用 `/api/save/list` 返回 `slots`
- 实际返回 `saves`（非 `slots`）
- 快速存档 `/api/save/quicksave` 返回 `code: None, msg: None`

### 🟡 #11 侦查(scout)无法指定目标

圣旨"侦查周边"被解析为 `scout` 指令，但因"侦查目标不存在"而失败。NLP应该将"周边"自动映射到相邻敌方地块，而不是报错。

### 🟡 #12 间谍目标"陈友谅"不存在

圣旨"派遣细作潜入陈友谅地盘"失败：`目标势力陈友谅不存在`。

这表明：
1. NLP识别到了"陈友谅"但映射到 `faction_chenyouliang` 时失败
2. 或者世界状态中该势力的状态有问题（troops=-1 导致被判定为已灭亡）

### 🟡 #13 回合推进速度异常快

带AI的回合推进仅耗时 1.3 秒，远低于预期的 60-90 秒。说明：
- AI推演（阶段③）可能因所有势力数据异常而全部走降级策略/跳过
- 没有实际的AI调用发生

## 五、根因分析

问题的核心链条：

```
势力 tiles=[] (数据为空)
  → troops=-1 (数据异常)
    → AI推演全部跳过（无有效势力可推演）
      → 回合推进 1.3s（无AI调用）
        → 无世界事件产生
          → 玩家感觉世界"死寂"
  
地块ID不一致 (tile_yingtian vs "18,12")
  → 地块操作全部失败
    → 行军/侦查/建造无法执行
  
军械不足 (80 vs 需要1000)
  → 征兵永远失败
    → 无法扩张
      → 死循环
```

## 七、建议修复优先级

1. **P0 - 修复势力tiles数据为空**：确保 `status` 和 `init` 返回的 `world_state` 一致
2. **P0 - 统一地块ID体系**：全部使用坐标格式（如"18,12"）或全部使用语义化ID，不要混用
3. **P0 - 修复资源字段**：`treasury`/`silver`、`reputation` 等字段统一，确保 `status` 不丢数据
4. **P1 - 调整初始军械/征兵成本**：确保玩家至少能成功征兵一次
5. **P1 - 修复 NPC势力 troops=-1**：确保AI推演能正常运行
6. **P2 - 增强战略建议内容**：传入足够上下文
7. **P2 - 统一存档接口**：`slots` → `saves`，`quicksave` 返回值
