# 2026-07-14

## 服务器稳定性修复（6项）
针对游玩过程中服务器失效问题，实施了以下修复：

### 1. `resolve_battle` 端点类型安全 (`api_server.py:2177`)
- 添加 `attacker_troops` / `defender_troops` 的强制 int 转换
- 防御前端传入 dict 导致 `TypeError: unsupported operand type(s) for *: 'dict' and 'float'`
- 添加兵力合法性校验（双方不能同时为0）

### 2. `execute_round` 分阶段异常隔离 (`round_engine.py:99`)
- 8 个阶段各自用独立 try/except 包裹
- 单阶段失败不再导致整个回合崩溃
- 失败阶段记录到 `summary["failed_phases"]` + `summary["partial_failure"]`
- 时间推进和权责统计也加了保护

### 3. LLM 客户端改进 (`hunyuan_client.py`)
- 使用 `API_TIMEOUT=90` 替代硬编码 120s
- 添加 httpx 连接池限制（max_keepalive=10, max_connections=30）
- `_call_api` 添加 `asyncio.wait_for` 外层超时兜底（API_TIMEOUT+30）
- 超时后自动进入 fallback_mode

### 4. 回合锁超时机制 (`round_engine.py:115`)
- `_round_lock` 从 `async with` 改为显式 `acquire/release`
- 获取锁时使用 `asyncio.wait_for(..., timeout=300)` 防止死锁

### 5. `advance_turn` 端点改进 (`api_server.py:1188`)
- 区分 `TimeoutError` 和一般异常
- 回合成功后清空 `_pending_commands`（之前未清理，可能跨回合重复执行）
- 失败时保留 `_pending_commands` 让玩家可重试

### 6. uvicorn 配置优化 (`api_server.py:7550`)
- `timeout_keep_alive=120` 防止连接挂死
- `limit_concurrency=50` 限制并发
- `limit_max_requests=1000` worker 定期重启防内存泄漏
- `backlog=128` 队列大小
