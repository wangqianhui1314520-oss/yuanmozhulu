## 二、方案A：Docker 一键部署（推荐，最稳定）

### 2.1 原理架构

项目内置多阶段 Dockerfile，CloudStudio 自动检测并构建：

```
Dockerfile 两阶段构建：

  Stage 1: node:20-alpine        Stage 2: python:3.11-slim
  ┌───────────────────┐          ┌──────────────────────────────┐
  │ npm ci             │          │  Supervisor (PID 1)           │
  │ vite build         │  拷贝    │  ├─ Nginx :80                 │
  │ → frontend/dist/   │─────────▶│  │   ├─ /api/* → 127.0.0.1:8800│
  └───────────────────┘          │  │   └─ /* → /app/frontend/dist│
                                 │  └─ Uvicorn :8800              │
                                 │      └─ python server.py        │
                                 └──────────────────────────────┘
```

### 2.2 操作步骤（5分钟）

**前置条件**：项目已推送到 Git 仓库（GitHub / Gitee / CNB）

**第1步：创建 CloudStudio 应用**

访问 [https://cloudstudio.net](https://cloudstudio.net) → 登录 → 点击「创建应用」→ 选择「从 Git 仓库导入」→ 粘贴仓库地址。

**第2步：等待自动构建**

CloudStudio 检测到根目录 `Dockerfile`，自动执行两阶段构建：
- 前端：`npm ci` → `vite build`（约1-2分钟）
- 后端：`pip install -r server/requirements.txt`（约1分钟）

**第3步：配置环境变量（关键！）**

构建完成后，进入容器终端：

```bash
# 方式一：写入 .env 文件（推荐）
echo 'TENCENT_API_KEY=你的CodeBuddy_API_Key' >> /app/.env

# 方式二：直接导出环境变量
export TENCENT_API_KEY="你的CodeBuddy_API_Key"
export PYTHONUTF8=1
export PYTHONIOENCODING=utf-8
```

> ⚠️ **`TENCENT_API_KEY` 为必填项**，缺少则 AI 战略/谍报/朝堂等所有 LLM 功能不可用，比赛评审将无法展示核心AI能力。

**第4步：重启容器**

CloudStudio 控制台 → 重启应用 → 等待 Supervisor 拉起 Nginx + Uvicorn（约10秒）。

**第5步：获取公网地址**

点击右侧「预览」面板，端口设为 **80**，获得公网URL。格式类似：
```
https://xxxxxxxxxxxx.cloudstudio.net
```

将此 URL 分享给评委即可直接体验游戏。

### 2.3 关键文件说明

| 文件 | 作用 |
|------|------|
| `Dockerfile` | 多阶段构建定义（Node.js 前端构建 + Python 后端运行） |
| `frontend/nginx.conf` | Nginx 反向代理规则：`/api/*` → Python, `/*` → 静态文件 |
| `supervisor.conf` | 进程管理器配置（同时管理 Nginx + Uvicorn） |
| `.dockerignore` | 排除 node_modules、日志、测试文件等，减小镜像体积 |

### 2.4 Docker 内服务端口规划

| 服务 | 内部端口 | 对外端口 | 说明 |
|------|---------|---------|------|
| Nginx | 80 | 80 | 反向代理 + 静态文件 + Gzip |
| Uvicorn (FastAPI) | 8800 | — | 仅内网，Nginx 转发 `/api/*` |

### 2.5 安全加固（已内置）

| 安全层 | 实现 |
|--------|------|
| 请求体限制 | `client_max_body_size 50M` |
| API 超时保护 | `proxy_read_timeout 120s`（AI 调用较慢） |
| 安全响应头 | `X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection` |
| Gzip 压缩 | text/css/js/json/svg |
| SPA 路由回退 | `try_files $uri /index.html` |
| 静态资源缓存 | `/assets/` 30天，`/data/` 7天 |

## 三、方案B：工作空间脚本部署（快速调试）

### 3.1 原理

不使用 Docker，直接在 CloudStudio 工作空间内执行脚本，依次完成依赖安装、前端构建、后端启动。FastAPI 同时提供 API（`:8800`）和前端静态文件（`StaticFiles` 挂载 `frontend/dist/`）。

### 3.2 操作步骤（8分钟）

**第1步**：同上，从 Git 仓库导入项目到 CloudStudio 工作空间。

**第2步**：打开终端，执行：

```bash
bash start_cloudstudio.sh
```

脚本自动执行：
1. `pip install -r server/requirements.txt` — 安装 11 个 Python 依赖
2. `cd frontend && npm install && npm run build` — 安装前端依赖 + Vite 构建
3. `python server.py` — 启动 FastAPI 服务

**第3步**：获取公网地址

CloudStudio 预览面板端口设为 **8800**，获得公网URL。

### 3.3 方案B注意事项

| 问题 | 说明 |
|------|------|
| 无 Nginx 反向代理 | FastAPI 直接暴露在公网，缺乏 Gzip/缓存/安全头 |
| 端口为 8800 | 与 Docker 方案的 80 端口不同 |
| 冷启动慢 | 每次重启需重新 `npm install`（可用 prebuild 缓存优化） |
| 环境变量 | 需在启动前 `export TENCENT_API_KEY=...` |
| 进程管理 | 单进程，崩溃不会自动重启（Docker 方案有 Supervisor） |

### 3.4 优化：添加 prebuild 脚本

在 CloudStudio 项目设置中，可配置 prebuild 命令缓存依赖：

```bash
# prebuild（每次工作空间创建时执行一次）
pip install -r server/requirements.txt
cd frontend && npm install
```

这样后续启动只需 `npm run build && python server.py`，缩短到约1分钟。

## 四、方案C：CloudBase 云托管部署（生产级）

### 4.1 原理

利用腾讯云 CloudBase 云托管服务，通过 `cloudbaserc.json` 配置自动部署 Dockerfile 构建的容器。适合需要固定域名、自动扩缩容、更高稳定性的场景。

### 4.2 操作步骤（10分钟）

**第1步**：登录 [CloudBase 控制台](https://console.cloud.tencent.com/tcb)

**第2步**：创建云开发环境（如已有则跳过）

选择「按量计费」→ 环境名称自定义 → 开通。

**第3步**：创建云托管服务

云托管 → 新建服务 → 选择「Dockerfile 部署」：
- 服务名称：`yuanmo-zhulu`
- 上传方式：选择项目根目录
- CPU：1核
- 内存：2GB
- 端口：80
- 副本数：1（最小）~ 1（最大）

**第4步**：配置环境变量

```json
{
  "TENCENT_API_KEY": "你的CodeBuddy_API_Key",
  "PYTHONUTF8": "1",
  "PYTHONIOENCODING": "utf-8"
}
```

**第5步**：部署

点击「部署」→ 等待构建（约3-5分钟）→ 获得固定域名：
```
https://yuanmo-zhulu-xxxxxxxxx.ap-shanghai.run.tcloudbase.com
```

### 4.3 方案C优势

| 特性 | 说明 |
|------|------|
| 固定域名 | 不随重启变化，适合比赛提交 |
| 自动扩缩容 | 支持流量高峰自动扩容 |
| HTTPS 自动配置 | 自带 SSL 证书 |
| 日志与监控 | 控制台查看请求日志、CPU/内存 |
| 环境变量管理 | 控制台面板配置，无需进终端 |

### 4.4 cloudbaserc.json 配置解读

```json
{
  "envId": "yuanmozhulu-d7g0odx41c77d74e6",
  "containerPort": 80,
  "dockerfile": "Dockerfile",
  "cpu": 1, "mem": 2,
  "minNum": 1, "maxNum": 1,
  "envVariables": {
    "TENCENT_API_KEY": "{{env.TENCENT_API_KEY}}"
  }
}
```

| 维度 | 方案A Docker | 方案B 脚本 | 方案C CloudBase |

| 部署难度 | ⭐ 低 | ⭐⭐ 中 | ⭐⭐ 中 |

| 域名固定性 | 不固定 | 不固定 | ✅ 固定域名 |

| HTTPS | ✅ 平台提供 | ✅ 平台提供 | ✅ 自带 |

```
开发阶段     →  方案B（快速调试，改完即看）
```

## 八、故障排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 首页白屏 | 前端未构建 | 确认 `frontend/dist/` 存在，执行 `npm run build` |
| AI 功能不可用 | API Key 未设置 | `echo $TENCENT_API_KEY` 确认；`.env` 是否在 `/app/` 下 |
| 地图不渲染 | 地图数据缺失 | 确认 `server/data/map/map_full.json` / `map_final.json` 存在 |
| `/api/health` 无响应 | 服务未启动 | `ps aux` 确认 uvicorn 进程存在 |
| Docker 构建失败 | 镜像拉取超时 | 重试；或改用方案B |
| 502 Bad Gateway | Nginx 连不上 Uvicorn | 等待 Supervisor 启动完成（startsecs=5），查看 `/app/logs/` |
| 内存不足 | 资源规格太低 | 升级到 2GB+ 内存（AI 模型调用需额外内存） |
| 安装依赖失败 | 网络问题 | `pip install -i https://mirrors.cloud.tencent.com/pypi/simple -r server/requirements.txt` |
